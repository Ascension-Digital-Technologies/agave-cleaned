# Code of Conduct

This repository follows the spirit of the open-source community standards used by many GitHub projects: be respectful, constructive, and focused on the work.

## Expected behavior

- Be professional and considerate.
- Keep discussion focused on the repository, documentation, scripts, and source review boundaries.
- Assume good intent while still asking for clear evidence and reproducible checks.
- Respect maintainers' time by using issues and pull requests with enough detail to act on.

## Unacceptable behavior

- Harassment, threats, or personal attacks.
- Posting private information without permission.
- Spam, trolling, or intentionally disruptive behavior.
- Publicly posting exploit details for security-sensitive issues.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues; lock conversations; or restrict participation when needed to keep the project safe and productive.

Security-sensitive issues should follow [`SECURITY.md`](SECURITY.md), not public issue discussion.
